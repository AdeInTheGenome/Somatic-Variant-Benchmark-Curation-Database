# Mosaic Somatic Variant Benchmark Curation Database

This repository contains an R Markdown workflow for constructing a multi-platform database of candidate mosaic somatic variants for use in benchmark development. The database integrates variant calls from a draft benchmark VCF with orthogonal support evidence from four sequencing technologies, producing a structured resource for manual curation and downstream benchmark refinement.

---

## Background

Mosaic variants — somatic mutations present in only a subset of cells — are among the most challenging variant types to benchmark accurately. Unlike germline variants, they occur at low variant allele frequencies (VAF), often below 5%, and require high-depth sequencing and rigorous orthogonal validation to distinguish true signal from sequencing noise.

This workflow was developed to support the curation of a draft mosaic benchmark derived from HG002 (NA24385), a well-characterized GIAB reference sample. The draft benchmark was generated using Strelka2 somatic variant calling, and this database is designed to assess each candidate variant's support across independent sequencing platforms before manual curation decisions are made.

---

## What This Workflow Does

The script performs the following steps:

**1. Benchmark variant annotation**
Loads the draft benchmark VCF and annotates each variant with:
- Variant category (`candidate` = PASS filter, `putative` = non-PASS)
- Variant type (SNV, indel, MNP)
- Strelka2-derived allele frequency (AF), computed separately for SNVs and indels using tier-aware read counts
- Genomic context flags: mappability, homopolymer region, germline indel overlap

**2. Duplex Sequencing support**
Downloads HG002 Duplex Sequencing mutation files and detects benchmark variants within them. Duplex Sequencing applies molecular barcoding to both strands of a DNA duplex, offering extremely low error rates and making it one of the most sensitive orthogonal validation methods available for low-VAF variants. For each variant, the workflow aggregates support across all available Duplex Sequencing runs and computes a 90% Wilson confidence interval on the mutation frequency.

**3. Orthogonal platform support (bam-readcount)**
For each of four additional sequencing platforms, the workflow reads pre-computed bam-readcount output and matches observed base counts to benchmark variant positions. Allele-specific depths and quality scores (mapping quality, base quality) are extracted and linked to each variant. Confidence intervals are computed per platform.

The four orthogonal platforms are:
- **BGI** — short-read sequencing
- **Element Biosciences** — short-read sequencing (standard and long read sets combined and analyzed separately)
- **PacBio HiFi** — long-read high-fidelity sequencing (Revio and Sequel instruments combined and analyzed separately)
- **Illumina** — short-read sequencing

**4. Database assembly**
All platform evidence is joined into a single variant-level database. Each row represents one benchmark variant and includes:
- Strelka2 AF and EVS score
- Per-platform depth, alt depth, mutation frequency, mapping quality, base quality, and Wilson CIs
- A `Detection_Category` field indicating which combination of platforms detected the variant
- Orthogonal CI flags comparing Strelka2 AF to the combined confidence interval across all platforms (with and without Duplex)
- Genomic context and germline indel overlap classifications

The final database is written to `scratch/mosaic_variant_database_RM.tsv`.

## Input Files

### Draft Benchmark VCF
`data/cand_put_mosaic_v0.0.3.FINAL.vcf`

A Strelka2-format somatic VCF containing candidate and putative mosaic variants called against HG002. Variants with `FILTER == PASS` are classified as `candidate`; all others are `putative`. The INFO and FORMAT fields follow Strelka2 conventions for SNVs and indels respectively.

### Duplex Sequencing Metadata
`data/dupseq-file-metadata.tsv`

A tab-separated file listing available Duplex Sequencing mutation files for HG002 samples, with columns including `File_type`, `sample`, and `url`. The workflow filters for `mut` file types from GIAB-HG002 samples and downloads them at runtime.

### bam-readcount TSVs
`scratch/bam_readcount/`

Pre-computed output from [bam-readcount](https://github.com/genome/bam-readcount) run against each platform's HG002 BAM at benchmark variant positions. Files are named by platform and variant type:

| File | Platform | Type |
|---|---|---|
| `BGI_RM_snv.tsv` | BGI | SNV |
| `BGI_RM_indel.tsv` | BGI | Indel |
| `Element_RM_snv_standard.tsv` | Element (standard) | SNV |
| `Element_RM_snv_long.tsv` | Element (long) | SNV |
| `Element_RM_indel_standard.tsv` | Element (standard) | Indel |
| `Element_RM_indel_long.tsv` | Element (long) | Indel |
| `Pacbio_RM_revio_snv.tsv` | PacBio Revio | SNV |
| `Pacbio_RM_revio_indel.tsv` | PacBio Revio | Indel |
| `Pacbio_RM_sequel_snv_A.tsv` | PacBio Sequel (run A) | SNV |
| `Pacbio_RM_sequel_snv_B.tsv` | PacBio Sequel (run B) | SNV |
| `Pacbio_RM_sequel_indel_A.tsv` | PacBio Sequel (run A) | Indel |
| `Pacbio_RM_sequel_indel_B.tsv` | PacBio Sequel (run B) | Indel |
| `Illumina_RM_snv.tsv` | Illumina | SNV |
| `Illumina_RM_indel.tsv` | Illumina | Indel |

## Output Database

`scratch/mosaic_variant_database_RM.tsv`

A tab-separated database with one row per benchmark variant. Key columns include:

| Column | Description |
|---|---|
| `CHROM`, `POS`, `REF`, `ALT` | Variant coordinates and alleles |
| `var_type` | SNV, indel, or MNP |
| `var_cat` | `candidate` (PASS) or `putative` |
| `AF` | Strelka2-derived allele frequency |
| `SomaticEVS` | Strelka2 empirical variant score |
| `VAF_Category` | Binned AF category (0%, 0–1%, 1–4%, etc.) |
| `SomaticEVS_Category` | Binned EVS category |
| `Genomic_Context` | Mappability and homopolymer region classification |
| `Germline_Indel_Check` | Whether the variant overlaps a known germline indel |
| `Detection_Category` | Which platforms detected the variant (e.g. `Illumina,Duplex,BGI`) |
| `{Platform}_depth` | Total read depth at position for each platform |
| `{Platform}_alt_depth` | Alt allele read count for each platform |
| `{Platform}_mut_freq` | Observed mutation frequency for each platform |
| `{Platform}_Lower_CI` / `_Upper_CI` | 90% Wilson CI on mutation frequency |
| `Orthogonal_Lower_CI` / `_Upper_CI` | Combined CI across all platforms |
| `Ortho_NO_DUP_Lower_CI` / `_Upper_CI` | Combined CI excluding Duplex Sequencing |
| `Ortho_CI_Flag` | Whether Strelka2 AF falls within combined orthogonal CI (`PASS`/`FAIL`) |
| `NonDup_CI_Flag` | Same check excluding Duplex Sequencing |

## Running the Workflow

Open `draft_mosaic_benchmark_with_RM.Rmd` in RStudio and click **Knit**, or run from the command line:

```bash
Rscript -e "rmarkdown::render('draft_mosaic_benchmark_with_RM.Rmd')"
```

The workflow is designed to be run from the repository root. File paths are managed with the `here` package and resolve relative to the `.Rmd` file location at `scratch/draft_mosaic_benchmark.Rmd`.

The output is an interactive HTML report with a floating table of contents, collapsible code blocks, and paginated summary tables at each stage. The final database TSV is written to `scratch/mosaic_variant_database_RM.tsv`.

## Methods Notes

**Allele frequency calculation for SNVs:** Strelka2 reports tier 1 and tier 2 read counts for each base (AU, CU, GU, TU). The workflow selects the tier specified by `TQSS_NT` and computes `AF = alt_tier_count / (alt_tier_count + ref_tier_count)`.

**Allele frequency calculation for indels:** Uses `TAR` (ref-supporting reads) and `TIR` (alt-supporting reads) fields, again tier-selected by `TQSI_NT`.

**Confidence intervals:** Wilson score intervals are computed using `Hmisc::binconf()` at α = 0.10 (90% CI) for per-platform frequencies and α = 0.02 (98% CI) for the combined orthogonal frequency. The combined frequency pools alt depth and total depth across platforms before computing the interval.

**Platform detection threshold:** A variant is considered detected by a platform if `alt_depth > 1` and mutation frequency is non-missing.
